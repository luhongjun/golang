# 让命令程序只执行下载动作，而不执行安装动作
go get -d example.com/pkg@v1.2.3

# 更新已有代码包及其依赖包。默认情况下，该命令只会从网络上下载本地不存在的代码包，而不会更新已有的代码包。
go get -u example.com/pkg@v1.2.3

# 只有在你包含了 -u 参数的时候才有效，不让 -u 去验证 import 中的每一个都已经获取了，这对于本地 fork 的包特别有用
go get -f example.com/pkg@v1.2.3

# 同时也下载需要为运行测试所需要的包
go get -t example.com/pkg@v1.2.3

# 允许使用不安全的 HTTP 方式进行下载操作
# 如果代码仓库（如公司内部的Gitlab）没有HTTPS支持，可以添加此标记。请在确定安全的情况下使用它。
go get -insecure example.com/pkg@v1.2.3

# 在获取源码之后先运行 fix 修正动作
go get -fix example.com/pkg@v1.2.3

# 显示操作流程的日志及信息，方便检查错误
go get -v example.com/pkg@v1.2.3
